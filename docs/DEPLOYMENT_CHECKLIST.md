## 🗄️ 数据库设置

### 第1步：创建数据库集合

在云开发控制台依次创建以下集合：

1. **users** - 用户表
   - 字段说明：
     - `_id`: 自动生成
     - `openid`: 用户唯一标识
     - `nickname`: 昵称（科学大师/小白）
     - `avatar`: 头像（存储首字符）
     - `grade`: 年级
     - `vip_level`: VIP等级（默认0）
     - `coins`: 金币数量（默认0）
     - `created_at`: 创建时间

2. **questions** - 题库表
   - 题目已准备在 [questions_sample.json](../database/questions_sample.json)

3. **wrong_questions** - 错题表
   - 字段说明：
     - `_id`: 自动生成
     - `user_id`: 用户ID
     - `question_id`: 题目ID
     - `wrong_answers`: 错误答案记录
     - `wrong_count`: 错误次数
     - `is_mastered`: 是否已掌握
     - `created_at`: 创建时间

4. **study_records** - 学习记录表
   - 字段说明：
     - `_id`: 自动生成
     - `user_id`: 用户ID
     - `date`: 日期（YYYY-MM-DD）
     - `question_count`: 答题总数
     - `correct_count`: 正确数量
     - `correct_rate`: 正确率
     - `study_duration`: 学习时长（秒）
     - `created_at`: 创建时间

### 第2步：配置数据库安全规则

**⚠️ 重要：开发阶段设置为宽松权限**

1. 打开云开发控制台
2. 点击"数据库" → "安全规则"
3. 为每个集合设置以下规则：

```json
{
  "read": true,
  "write": true
}
```

**安全提示**：上线前需要修改为严格规则：
```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

### 第3步：导入题目数据

1. 在云开发控制台点击"数据库" → "questions"集合
2. 点击"导入"按钮
3. 选择文件：[database/questions_sample.json](../database/questions_sample.json)
4. 确认导入（应显示20道题目）

或者手动添加几道测试题目：

```json
{
  "subject": "物理",
  "grade_level": "小学四年级",
  "type": "判断题",
  "question": "水星上有水吗？",
  "answer": "错",
  "options": ["对", "错"],
  "difficulty": 1,
  "knowledge_point": "太阳系",
  "explanation": "水星名字虽有'水'，但表面没有液态水",
  "status": "active"
}
```

## ☁️ 云函数部署

### 云函数清单（共6个）

| 序号 | 云函数名 | 功能 | 状态 |
|------|----------|------|------|
| 1 | login | 用户登录获取openid | ⬜ 待部署 |
| 2 | getUserInfo | 获取用户信息 | ⬜ 待部署 |
| 3 | updateUserInfo | 更新用户信息 | ⬜ 待部署 |
| 4 | getQuestion | 获取题目列表 | ⬜ 待部署 |
| 5 | submitAnswer | 提交答案并记录 | ⬜ 待部署 |
| 6 | getWrongQuestions | 获取/管理错题 | ⬜ 待部署 |

### 部署步骤

**方法一：逐个部署（推荐）**

1. 在微信开发者工具左侧目录找到 `cloudfunctions` 文件夹
2. 右键点击每个云函数文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待部署完成（每个约10-30秒）

**方法二：批量部署**

1. 右键点击 `cloudfunctions` 文件夹
2. 选择"同步云函数列表"
3. 然后逐个右键部署

### 部署验证

部署完成后，在云开发控制台检查：

1. 点击"云函数"
2. 应该能看到所有6个云函数
3. 状态显示为"正常"

## 🐛 常见问题排查

### 问题1：云函数调用失败

**错误信息**: `FunctionName parameter could not be found`

**解决方案**:
1. 检查云函数是否已部署
2. 在云开发控制台确认云函数存在
3. 重新部署云函数

---

### 问题2：数据库查询失败

**错误信息**: `查询参数对象值不能均为undefined`

**解决方案**:
1. 检查数据库安全规则是否已设置
2. 确认数据库集合已创建
3. 检查集合中是否有测试数据

---

### 问题3：用户不存在

**错误信息**: `用户不存在`

**解决方案**:
1. 首次运行时需要选择身份创建用户
2. 检查updateUserInfo云函数是否正常
3. 在数据库中手动添加一条用户记录测试

---

### 问题4：获取不到题目

**错误信息**: `暂无题目`

**解决方案**:
1. 确认questions集合中已导入题目数据
2. 检查题目数据的grade_level字段是否与用户年级匹配
3. 确认题目的status字段为"active"

---

### 问题5：云函数日志查看

**如何查看日志**:
1. 打开云开发控制台
2. 点击"云函数"
3. 点击具体云函数名称
4. 查看"日志"标签页
5. 可以按时间筛选查看最新日志

## 📊 云函数代码结构

```
输入: {}
输出: { openid, appid, unionid }
```

```
输入: { openid }

```

