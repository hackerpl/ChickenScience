## 🚀 正式部署步骤

### 步骤 1: 开通云开发

1. 打开微信开发者工具
2. 点击顶部菜单 **"云开发"**
3. 点击 **"开通"** 按钮
4. 按提示完成开通（免费额度）

### 步骤 2: 获取环境ID

1. 在云开发控制台页面
2. 找到 **"环境ID"**（格式类似：`cloud1-xxxxxxxxxxxxxx`）
3. 复制这个ID

### 步骤 3: 配置环境ID

打开 `miniprogram/app.js`，取消注释并配置：

```javascript
if (wx.cloud) {
  wx.cloud.init({
    env: 'cloud1-6gad1q0255f7510b', // 替换为你的环境ID
    traceUser: true
  })
}
```

### 步骤 4: 部署云函数

在微信开发者工具中：

1. 右键点击 `cloudfunctions` 文件夹
2. 选择 **"上传并部署：云端安装依赖"**
3. 等待部署完成（所有云函数图标变成正常状态）

### 步骤 5: 创建数据库集合

1. 在云开发控制台点击 **"数据库"**
2. 点击 **"添加集合"**
3. 创建以下集合：
   - `users`
   - `questions`
   - `wrong_questions`
   - `study_records`

### 步骤 6: 添加测试题目

1. 进入 `questions` 集合
2. 点击 **"添加记录"**
3. 添加以下JSON数据：

```json
{
  "subject": "物理",
  "grade_level": "小学四年级",
  "type": "判断题",
  "question": "水星上有水吗？",
  "answer": "错",
  "options": ["对", "错"],
  "difficulty": 1,
  "knowledge_point": "太阳系",
  "status": "active"
}
```

### 步骤 7: 启用真实数据模式

完成上述步骤后，在以下文件中：

1. `app.js` - 取消注释云初始化代码
2. `pages/index/index.js` - 取消注释真实请求代码（约83行开始）
3. `pages/study/study.js` - 取消注释真实请求代码（约137行开始）

## 🔧 常见问题

### Q: 提示"getAppConfig error with backend: timeout"
**A:** 这是云开发环境ID配置问题。当前已禁用云初始化，不影响模拟模式使用。

### Q: 如何知道云函数是否部署成功？
**A:** 云函数文件夹图标会从云朵状变成正常状态，无红色感叹号。

### Q: 模拟数据和真实数据可以同时使用吗？
**A:** 可以。当前代码优先使用模拟数据，部署云函数后，取消注释真实请求代码即可切换。

### Q: 如何添加更多题目？
**A:**
1. 正式部署：在云开发控制台的 `questions` 集合中添加
2. 模拟模式：在 `pages/study/study.js` 的 `mockQuestions` 数组中添加
